# TESTING_GUIDE.md
Руководство по тестированию SM Codex Portal

---

# 1. Smoke-тесты

Каждый сборочный релиз должен проходить:

- переход по всем разделам
- открытие всех модалок
- переключение subnav вкладок
- открытие всех квизов
- открытие всех калькуляторов

---

# 2. Функциональные тесты

## 2.1 Калькуляторы
Протестировать:

- корректность формул
- корректность ID
- реакцию на изменения полей
- пересчёт в реальном времени

## 2.2 Квизы
- загрузка JSON
- выбор вариантов
- отображение результатов

## 2.3 Навигация
- hash-навигация
- sticky-паттерны
- состояние subnav

---

# 3. UX-тесты

- скорость загрузки
- читаемость блоков
- отсутствие визуального шума
- правильность отступов
- проверка мобильной версии

---

# 4. Кросс-браузерные тесты

Обязательно проверить в:

- Chrome  
- Firefox  
- Safari (если доступно)

---

# 5. Ошибки

Любая ошибка:
1. воспроизводится,  
2. описывается,  
3. фиксируется в TROUBLESHOOTING.md,  
4. исправляется в коде,  
5. добавляется в CHANGELOG.
