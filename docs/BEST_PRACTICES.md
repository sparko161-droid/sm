# BEST_PRACTICES.md
Лучшие практики разработки SM Codex Portal

---

# 1. Архитектурные

- использовать модульную архитектуру
- выносить общие функции в core
- держать калькуляторы в одном месте
- использовать один шаблон модального окна
- избегать логики в HTML

---

# 2. UI/UX практики

- единый стиль карточек и секций
- sticky-паттерны для разделов
- избегать перегрузки текстом
- структурировать контент через секции
- использовать короткие списки

---

# 3. DevOps / сопровождение

- обновлять CHANGELOG при каждом релизе
- поддерживать документацию актуальной
- фиксировать ошибки в TROUBLESHOOTING.md
- регламентировать добавление новых кейсов

---

# 4. Контент

- любой материал должен иметь автора
- кейсы → по шаблону CASE_TEMPLATE.md
- онбординги → краткие, понятные, проверяемые
- квизы → только после внедрения материалов

---

# 5. Коммуникации

- PR обсуждаются внутри команды
- важные правки → в PLAN.md
- спорные решения → в INDEX.md или ARCHITECTURE.md
