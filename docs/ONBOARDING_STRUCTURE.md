# ONBOARDING_STRUCTURE.md
Структура всех онбордингов SM Codex Portal

Документ определяет стандарты оформления онбордингов:
- для поддержки (L1/L2/L3),
- для внедрения,
- для продаж,
- для других будущих отделов.

---

# 1. Принципы онбординга

1. **Структура 3–4 блоков слева-направо**  
   (Месяц 1 → Месяц 2 → Месяц 3 → при необходимости Месяц 4)

2. **Каждая карточка = цель месяца**

3. **Формат контента:**
   - краткий текст,
   - списки,
   - без перегрузки длинными абзацами.

4. **Должен показывать прогресс обучения**

---

# 2. Стандартная карточка онбординга

```html
<article class="card onboarding-card">
  <h3>1-й месяц</h3>
  <ul>
    <li>Понимание процессов отдела</li>
    <li>Изучение регламентов</li>
    <li>Прохождение базовых квизов</li>
  </ul>
</article>
````

---

# 3. Принципы оформления

* Заголовок = «1-й месяц», «2-й месяц», «3-й месяц»
* Содержимое = 4–8 пунктов
* Каждый пункт = измеряемая цель

---

# 4. Требования к онбордингу

* должен описывать реальные цели обучения
* должен быть связным с регламентами
* должен ссылаться на кейсы и квизы
* должен быть проверяемым руководителем

---

# 5. Требования по обновлению онбординга

Обновляется:

* при добавлении новых кейсов
* при изменении регламентов
* при изменении требований по грейдам

Ответственные: тимлиды отдела + разработчики
