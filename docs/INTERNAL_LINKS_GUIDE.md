# INTERNAL_LINKS_GUIDE.md
Рекомендации по внутренним ссылкам SM Codex Portal

Документ определяет, как оформлять переходы между кейсами, разделами, квизами и регламентами.

---

# 1. Типы внутренних ссылок

## 1.1 Ссылки между разделами
Используют hash-навигацию:

```html
<a href="#support">Перейти в поддержку</a>
````

## 1.2 Ссылки внутри раздела

```html
<a href="#support?section=l1">Перейти к L1</a>
```

## 1.3 Ссылки на квизы

```html
<span class="quiz-trigger" data-quiz-category="support" data-quiz-id="routing">
  Пройти квиз
</span>
```

## 1.4 Ссылки на кейсы

Рекомендуется использовать карточки, а не text-link.

---

# 2. Как правильно оформлять перекрёстные ссылки

## Пример:

В кейсе L2 можно дать ссылку:

```
См. также: Support → Матрица компетенций → раздел L2
```

## Пример:

В онбординге внедрения:

```
Рекомендуется пройти: Квиз по этапам внедрения
```

---

# 3. Чего нельзя делать

❌ Использовать относительные пути (`../support`)
❌ Ставить ссылки на HTML-файлы из `/sections`
❌ Создавать ссылки на старые версии страниц
❌ Переопределять логику Router

---

# 4. Рекомендации по контенту

* ссылка всегда указывает на **конкретный раздел**, а не просто на страницу
* ссылки располагаются внизу кейса или блока
